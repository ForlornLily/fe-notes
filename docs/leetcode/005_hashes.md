# 非顺序结构：字典和散列表

以 key, value 的形式存储值。

相对于集合，集合以 value, value 的形式

即 ES6 里面 Set 和 Map 的区别

字典也称为映射

## 字典

字典的实现略过，总体和集合一致，只不过多了个 key

`add("Saber")`变成`add("first", "Saber")`

## 散列表/哈希表（hash table/hash map）

字典的一种表现

通过计算一个关于键值的函数，将所需查询的数据映射到表中一个位置

映射函数称做散列函数，存放记录的数组称做散列表。

简而言之就是通过 key 访问 value

### 用 ES6 的 Map 实现即可，跳过

### 散列表复杂度

插入、删除、搜索为 O(n)
